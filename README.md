# Automated CI/CD DevOps Pipeline with Jenkins, SonarCloud & Render

An automated, containerized Continuous Integration and Continuous Deployment (CI/CD) architecture. This repository establishes a multi-stage software delivery pipeline triggered automatically via source control changes, executing security scans, container assembly checks, and production deployment.

---

## 🛠️ Pipeline Architectures & Lifecycle Stages

This orchestration workflow is configured across two distinct pipeline structures depending on deployment requirements:

### Pipeline 1: Full Distribution (With Registry Push)
Designed for automated testing, public registry distribution, and instant cloud deployment.
1. **Fetch Source Code**: Monitored via Git SCM, pulling fresh trees on webhook execution.
2. **Code Quality Analysis**: Connects code metrics directly to SonarCloud using the `sonar-scanner-cli` engine container.
3. **Vulnerability Scanning**: Executes filesystem level dependency and secret compliance auditing using an `aquasec/trivy` container.
4. **Build & Push Docker Image**: Compiles your code using a native `Dockerfile`, tags it dynamically, logs into Docker Hub securely, and pushes the production image to the public container registry.
5. **Deploy to Render**: Issues an authenticated HTTP POST request to Render's Deploy Webhook to pull the latest repository changes and update production instantly.

### Pipeline 2: Lean Infrastructure (Local Build Verification Only)
Optimized for direct repository-to-cloud delivery without creating an external registry footprint.
* Follows the exact same stages as Pipeline 1, but **Stage 4 (Build Docker Image)** compiles and verifies the container layer strictly locally on the Jenkins server environment. It completely bypasses the Docker Hub login and registry push requirements, saving bandwidth and network overhead.

---

## 📋 Architectural Integration Checklist

To replicate or maintain this pipeline structure across clean target environments, verify the following configuration blocks:

- [ ] **Active Communication Tunnel**: Local automated builds require an open `ngrok http 8080` background tunnel forwarding to the local Jenkins listener.
- [ ] **GitHub Webhook Routing**: Configured inside repository settings (`/settings/webhooks`) pointing to the active ngrok URL ending in `/github-webhook/` with the payload format explicitly set to `application/json`.
- [ ] **Jenkins Automated Build Triggers**: The target pipeline job must have the checkbox for **"GitHub hook trigger for GITScm polling"** actively checked.
- [ ] **Global Jenkins Credentials Matrix**:
  * `SONAR_TOKEN` (Secret Text) - Standard token verification mapping to your SonarCloud account.
  * `docker-hub-credentials` (Username with Password) - Standard secure credentials for automated registry pushes.
  * `RENDER_DEPLOY_HOOK` (Secret Text) - The unique secret deploy hook string generated by your specific Render service dashboard.
